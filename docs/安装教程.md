# 零基础安装教程（Windows 手把手）

从没写过代码、没装过 Python 也没关系。跟着下面一步步来，大概 10 分钟就能把一堆发票跑成一张 Excel。

每一步都很短，做完一步再做下一步。哪一步报错了，翻到最后的「常见报错对照表」对号入座。

---

## 你会用到

- 一台 Windows 电脑
- 一个能读图的大模型 key（阿里云百炼 / 智谱，注册几分钟拿到，有免费额度）——**只有识别图片/扫描件才用到**；纯电子发票 PDF 不需要 key 也能跑

---

## 第 1 步：装 Python

1. 打开 Python 官网 <https://www.python.org/downloads/>，点黄色大按钮下载最新版。
2. 双击安装。**第一屏最下面有个勾选框 `Add python.exe to PATH`，一定要勾上**，再点 `Install Now`。
3. 装完，按 `Win` 键搜「cmd」打开黑色命令行窗口，输入下面这行回车：

```bash
python --version
```

出现 `Python 3.x.x` 就成功了。如果提示「不是内部或外部命令」，说明第 2 步的勾没勾上，卸载重装、记得勾 PATH。

---

## 第 2 步：下载代码

1. 打开项目主页，点绿色 `Code` 按钮 → `Download ZIP`。
2. 把下载的 zip 解压，得到一个 `fapiao2excel` 文件夹。
3. 进到这个文件夹，**在顶部地址栏点一下，把路径删掉、输入 `cmd` 回车**——会弹出一个已经定位在项目里的命令行窗口。后面的命令都在这个窗口里输。

---

## 第 3 步：装依赖

在上一步的命令行窗口里输入回车：

```bash
pip install -r requirements.txt
```

它会自动装一堆东西，等它停下来。

如果卡很久或提示超时，换国内源再来一次：

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

---

## 第 4 步：配置 key（只识别图片/扫描件才需要）

> 如果你的发票**都是电子发票 PDF**，这一步可以先跳过，工具会直接精确解析、不用 key。手头有图片或扫描件的，再配 key。

1. 文件夹里有个 `.env.example`，复制一份，改名成 `.env`（前面那个点别丢）。
2. 用记事本打开 `.env`，填三行：

```
API_KEY=你的key
BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
VL_MODEL=qwen-vl-max
```

没有 key 的话：注册阿里云百炼（或智谱），在控制台建一个 API-KEY，新用户有免费额度。不同模型的 `BASE_URL / VL_MODEL` 见 README 的「用哪个模型」表。

---

## 第 5 步：放发票，跑起来

1. 把要识别的发票（PDF 或图片都行）丢进 `samples` 文件夹。
2. 命令行里输入回车：

```bash
python src/extract.py samples -o out/票据明细.xlsx
```

跑完，打开 `out` 文件夹里的 `票据明细.xlsx`——一堆发票就整整齐齐落成一张表了。

## 常见报错对照表

| 报错 / 现象 | 原因 | 怎么解决 |
|---|---|---|
| `'python' 不是内部或外部命令` | 装 Python 没勾 PATH | 重装勾上 `Add python.exe to PATH`，或用 `py` 代替 `python` 试 |
| `ModuleNotFoundError: No module named xxx` | 依赖没装 / 不在项目文件夹里装 | 确认命令行在 `fapiao2excel` 文件夹里，重跑第 3 步；不行用 `python -m pip install -r requirements.txt` |
| `pip` 超时 / 失败 | 默认源慢 | 用第 3 步的清华源那行 |
| 缺 `API_KEY` / 401 / 认证失败 | `.env` 没配好 | 回第 4 步，确认 `.env`（不是 `.env.example`）里 key 填对了 |
| 装了好几个 Python，还报缺包 | 多版本环境串了 | 建干净虚拟环境：`python -m venv venv` → `venv\Scripts\activate` → 再跑第 3 步 |

> 排查顺序：① `python --version` 出版本 → ② 装依赖不报错 → ③ `.env` 填好（用图片才需要）→ ④ 跑第 5 步。
> 还是卡住，把**完整报错原文**贴到 issue，别只说「跑不了」，报错原文最好定位。
